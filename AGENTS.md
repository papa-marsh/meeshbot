# meeshbot AGENTS.md

GroupMe bot built with FastAPI + Oxyde ORM. Receives webhook events from GroupMe groups, persists messages, and dispatches slash commands.

FUTURE: Will be extended to include LLM-based chat-bot resopnses.

## Environment

- Uses **uv** for dependency management. 
- Do not use `pip` directly
- Use `uv run <command>` instead of activating venv

```bash
uv add <package>            # add runtime dep
uv add --dev <package>      # add to [dependency-groups.dev]
uv sync                     # install all deps
uv run <command>            # run command in venv
```

## Running Locally

Runs in Docker Compose. The `meeshbot/` directory is volume-mounted, so code changes hot-reload without a rebuild.

```bash
docker compose up -d --build   # initial start
make deploy                    # rebuild + restart + tail logs
make logs                      # tail logs
make shell                     # IPython shell inside the running container
```

## Linting & Types

```bash
uv run mypy meeshbot           # type check
uv run ruff check meeshbot     # lint
uv run ruff format meeshbot    # format
```

Both ruff and mypy are configured to exclude `meeshbot/migrations/`.

Strict mypy config — all functions must be fully typed.

## Oxyde ORM

Django-style, type-safe, Pydantic-centric asynchronous ORM ([Docs](https://github.com/mr-fatalyst/oxyde)).

- Config is in `oxyde_config.py` at the repo root (not inside the package)
- Use existing model definitions and query patterns as examples when adding or extending DB operations
- Register new models in `oxyde_config.py` under `MODELS`
- Migration files and `.pyi` stubs are auto-generated by running `uv run oxyde makemigrations`
- Migrations land in `meeshbot/migrations/`

Oxyde is still in alpha and while it isn't owned by the user, its developer is responsive to PRs. Simple changes and bugfixes can be committed and pushed via the fork located at `~/Repositories/oxyde`.

**Query patterns:**
```python
await MyModel.objects.all()
await MyModel.objects.filter(field=value).all()
await MyModel.objects.get(id=1)
obj, created = await MyModel.objects.get_or_create(id=x, defaults={...})
obj.field = new_value
await obj.save()
await obj.delete()
```

DB connection is managed via `db.lifespan()` in `app.py` — no manual `db.init()` needed.

## GroupMe

Messages are posted to GroupMe using a **bot ID**. Each bot is associated with its group according to the mappings in `meeshbot/integrations/groupme/secrets.py`.

```python
# Post a message to a group
await GroupMeClient().post_message(group_id=webhook.group_id, text="hello")
```

Webhook auth is a `?token=` query param checked against `GROUPME_WEBHOOK_TOKEN` in `.env`.

## Adding a Command

1. Create `meeshbot/commands/<name>.py`:
```python
from meeshbot.integrations.groupme.client import GroupMeClient
from meeshbot.integrations.groupme.types import GroupMeWebhookPayload

async def mycommand(webhook: GroupMeWebhookPayload) -> None:
    await GroupMeClient().post_message(group_id=webhook.group_id, text="response")
```

2. Register it in `meeshbot/commands/registry.py`:
```python
from meeshbot.commands import mycommand

COMMAND_REGISTRY: dict[str, CommandFuncT] = {
    ...
    "/mycommand": mycommand,
    "/mycommand-admin": admin_only(mycommand),  # if admin-only
}
```

Commands are triggered by messages starting with `/`. The `admin_only` decorator gates on `ADMIN_USER_IDS` in `secrets.py`.
